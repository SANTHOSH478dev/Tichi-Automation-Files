DR-001 Application allows login with invalid email format — no app-level inline validation Login High High Open TC-LG-05
Description: The Login page does not validate email format at the application level. Inputs like userexample or @example.com only trigger a browser-native HTML5 tooltip — no custom inline error UI is rendered. DevTools Console confirms zero app-level validation errors.
Expected: Custom inline error: "Please enter a valid email address." — login request blocked.
Actual: Only browser HTML5 tooltip appears. App relies entirely on browser-native validation with no fallback.
Suggested Fix: Add JS regex validation on submit and render a styled inline error matching the app's design system (as seen on the Sign Up page).
Test Data: userexample, @example.com, user@, user@.com, user@@domain.com
